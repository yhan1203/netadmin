# ADR-001: 为什么选 Netmiko 而不是 NAPALM

## 背景
需要选择一个底层网络设备连接库，要求同时支持华为和思科交换机。

## 选项

| 维度 | Netmiko | NAPALM |
|------|---------|--------|
| 华为支持 | ✅ 原生支持 `huawei` device_type | ⚠️ 需要 `napalm-huawei` 第三方驱动 |
| 思科支持 | ✅ `cisco_ios`、`cisco_xe` 等 | ✅ `ios`、`ios-xe` |
| 安装 | 轻量，依赖少 | 较重，依赖较多 |
| 灵活性 | 直接发命令，自由度高 | 抽象层高，受限但统一 |
| 社区 | 活跃（Kirk Byers 维护） | 稳定但更新慢 |
| 学习成本 | 低，SSH + 发命令 | 中，需要理解 getter 体系 |

## 决策
选 **Netmiko**。

## 理由
1. 华为支持更成熟，原生 device_type 即可用
2. 轻量无外部依赖，适合 CLI 工具
3. 直接发命令的方式更适合"照猫画虎"场景——我们需要抓取原始配置做模板，不需要 NAPALM 的抽象
4. 社区活跃，遇到问题容易找到解决方案

## 后果
- 正向：快速开发，灵活度高
- 代价：需要自己处理厂商差异（通过 commands.py 映射层）
- 缓解：commands.py 的映射模式未来可以轻松切换到 NAPALM 如果需求变化
