# 需求文档 — P001-netadmin

## 目标
一个开箱即用的命令行工具，统一管理华为和思科交换机。解决运维"同时管两个厂商设备"时工具不统一、需要记两套命令的痛点。

## 用户画像

| 角色 | 痛点 | 使用频率 |
|------|------|---------|
| 驻场运维 | 同时管华为和思科，每天记两套命令 | 每天多次 |
| 网络工程师 | 批量部署新设备，配置标准化 | 项目式 |
| IT 主管 | 要做安全合规审计，看整体健康状态 | 每周 |
| 新人/实习生 | 交接时没有设备清单，不熟悉环境 | 入职时 |

## 用户场景

- **早上一看：** `netadmin check --all` 巡检所有设备健康状态
- **交接清单：** `netadmin scan 192.168.1.0/24` 发现网段里所有设备
- **批量上线：** `netadmin learn 10.0.1.1 -o t.yaml` 学标准配置 → 改模板 → `netadmin apply t.yaml -d 10.0.1.2`
- **安全合规：** `netadmin audit --all` 出全量审计报告
- **配置回档：** `netadmin backup run` 每天备份，出事后 `netadmin backup diff 1 2` 看谁改了啥
- **故障排查：** `netadmin interface list 10.0.1.1` 看接口状态，`netadmin exec 10.0.1.1 "display logbuffer"` 看日志

## 功能列表（MVP）

### P0 — 必须要有
- [x] SSH 连接华为/思科（connector）
- [x] 厂商命令映射（commands）
- [x] 配置备份 + 版本管理（backup）
- [x] 照猫画虎（learn → apply）
- [x] 网段设备发现（scan）
- [x] 健康检查（check）
- [x] 安全审计（audit）
- [x] VLAN 管理（vlan）

### P1 — 最好有
- [ ] 配置回滚（rollback）
- [ ] LLDP/CDP 拓扑发现（topology）
- [ ] 定时备份（schedule backup）
- [ ] 钉钉/Telegram 告警（notify）

### P2 — 锦上添花
- [ ] Web 仪表盘
- [ ] 配置模板市场

## 非功能需求
- **兼容：** 华为 S/CE 系列 + 思科 Catalyst/IOS-XE
- **部署：** `pip install` 一键安装，零外部依赖
- **安全：** 密码不落盘，支持环境变量注入
- **性能：** 50 台设备并发备份 < 2 分钟

## 不做
- 不支持图形界面 v1（以后做 Web）
- 不支持 H3C/锐捷（以后扩展）
- 不替代 Netmiko（底层用它，面向运维而不是开发者）
