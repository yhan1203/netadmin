# 设计文档 — P001-netadmin

## 架构概览

```
用户终端
    │
    ▼
┌─────────────────────────────┐
│  CLI (Click + Rich)         │  ← 用户交互层
│  13 个命令，彩色终端输出     │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│  commands.py                │  ← 语义映射层
```

## 模块说明

| 模块 | 职责 | 关键依赖 |
|------|------|---------|
| `cli.py` | Click 命令定义、Rich 输出、参数解析 | Click, Rich |
| `connector.py` | SSH 连接管理、厂商识别、Netmiko 封装 | Netmiko, Paramiko |
| `commands.py` | 语义命令 → 厂商命令映射 | 无 |
| `backup.py` | 配置备份、SQLite 版本管理、diff | SQLite3 |
| `learn.py` | 从设备学配置、结构化提取、YAML 生成 | re, yaml |
| `apply.py` | YAML 模板 → 设备配置命令 | re, yaml |
| `vlan.py` | VLAN 管理（查看/创建/删除/端口分配） | — |
| `interface.py` | 接口状态查询 | — |
| `scanner.py` | 网段扫描、端口探测、厂商识别 | socket, subprocess |
| `checker.py` | 健康检查（CPU/内存/温度）、安全审计 | — |
| `config.py` | YAML 配置加载、设备清单管理 | yaml |

## 技术选型

| 选项 | 方案 | 理由 |
|------|------|------|
| 网络库 | Netmiko | 华为+思科都支持，社区成熟，封装了 SSH 细节 |
| CLI 框架 | Click | Python 生态最成熟的 CLI 框架，子命令支持好 |
| 终端 UI | Rich | 表格/面板/语法高亮，运维看了觉得专业 |
| 配置格式 | YAML | 运维熟悉，可读性好，支持注释 |
| 版本存储 | SQLite | 零部署成本，嵌入式，够用 |
| 测试 | pytest | 社区标准 |
| 打包 | setuptools | pip install 即可安装 |

## 为什么选 Netmiko 而不是 NAPALM

| 维度 | Netmiko | NAPALM |
|------|---------|--------|
| 华为支持 | ✅ 成熟 | ⚠️ 有限 |
| 安装复杂度 | 轻量 | 需要更多依赖 |
| 灵活性 | 直接发命令，可控 | 抽象层高，受限 |
| 社区活跃度 | 高 | 一般 |

**结论：** Netmiko 更轻、华为支持更好，适合 CLI 工具。

## 安全设计

- 密码通过 YAML 或环境变量传入，不硬编码
- 备份文件存本地，不自动上传第三方
- 审计报告不包含明文密码
- `.gitignore` 排除备份目录和数据库
